    Git y GitHub

    Creando un repositorio local

    Un repositorio no es nada más que un directorio, una carpeta o contenedor de archivos en el cual almacenaremos nuestro trabajo

    Elegida una ubicación en nuestro espacio de almacenamiento, crea una carpeta con el nombre que quieras. Bien, presta atención a este atajo, dando clic derecho a la carpeta creada, hasta aquí vacia, y selecciona abrir en terminal.

    Sucedera que se abrira una ventana de terminal de Windows Powerhell, no te preocupes si no la conoces, pues sólo deberas escribir "code ." (code tecla espacio y signo punto).

    Bienvenido a Visual Studio Code donde puede ver el Explorador, la carpeta con el nombre que has puesto pero esta vacío, comienza a llenarla.

    Crearemos un "nuevo archivo" con el nombre index.html, el cual será nuestra página principal y también un archivo con el nombre Readme.md, que es un archivo texto para que escribar la descripción de tu proyecto

    Git local

    Paso siguiente, utilizaremos la terminal desde el Visual Studio Code, oprimiendo Crtl + ñ.

    Esta terminal automaticamente se posiciona en la ubicación de nuestra carpeta, es muy importante prestar atención en ese posicionamiento, debe ser el mismo en el que estamos trabajando.

    En la terminal abierta, escribiremos los siguientes comandos:

    git init: inicializa la funcionalidad del git, creando una carpeta oculta con el nombre de .git, es necesario recomendarte que no toques esos archivos, no los borres ni los cambies de ubicación, no toques éso!.

    Como será nuestro repositorio, lo registramos con nuestros datos, utilizando el comando "git config --global user.name "nombre de ejemplo"", de la misma manera repetimos para "git commit config --global user.email "email@deprueba.com"".

    Creando repositorio en GitHub:

    Una vez ingresado en la cuenta creada en la página oficial de GitHub, a la parte superior del margen derecho, se puede desplegar la opción de "New repository".

    Antes de darle clic a Create repository, deberás completar los campos en blanco, comenzando por el nombre de repositorio, si quieres, puedes agregar una descripción, luego seleccionar el estado que tendrá el proyecto, puede ser público o privado, como también puedes o no iniciar agregando un archivo README.

    Listo, así como el Shampoo lleva instrucciones, no queda más que seguir las mismas que ofrece la página de GitHub.

    Como ya hemos hecho un git init, ya tenemos nuestro archivo README.md, sólo queda por hacer un "git add .", y un "git commit -m "el nombre o detalle de lo que has hecho que referencie a los cambios"; de esta manera agregaremos todos los archivos al directorio .git. Ahora si, pasaremos al siguiente apartado que menciona la página y escribiremos "git remote add origin https://github.com/tunombreusuario/tunombrerepositoriocreado (bastante largo? utiliza el copy paste!), de esta manera se conectará con el repositorio de GitHub al cual enviaremos nuestros archivos, mediante el uso del comando "git push -u origin master".

    Compruébalo, tanto tu carpeta local, como el repositorio de GitHub tienen los mismos archivos.

    Creando una rama de desarrollo

    También conocida como branch, crearemos otra rama a la cual iremos modificando, es así que en nuestra terminal escribiremos "git branch develop", la famosa rama de desarrollador en la que realizaremos todos nuestros cambios hasta que esten listos para ser publicados en nuestra rama master.

    Si te arrepientes puedes eliminarla de la misma manera escribiendo "git branch -D develop", si no, continuemos. Agrega los cambios y realiza un commit, para luego hacer uso del push con el siguiente comando "git push --set-upstream origin develop".

    Antes de seguir avanzando, haremos un salto gigante, una vez finalizado y aplicado los cambios que queremos a nuestra rama develop, vamos a fusionar nuestros arvhicos con la rama principal del proyecto, para ellom, devemos posicionarnos en nuestra rama principal, consulta mediante el comando "git branch", posisiónate mediante "git checkout master", y luego se utiliza el siguiente comando: "git marge develop".

    Ahora si, entra a tus archivos html y modifícalos, carga información, usa frameworks, aplícales tus propios estilos mediante css, juega!.
